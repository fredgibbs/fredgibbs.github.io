Introduction

Local Observations

  • My history classes (not just DH ones) have tended toward digital public history projects
  • Building is more fun than grading (and teaches more)
  • Engaging students with ongoing projects is better than fantasizing about sustainability plans

Wider Observations

  • More instructors are having students make websites or contribute to digital projects
  • More instructors want to learn how to gain more control and aggregate work over time
  • Wild WWWest with little standardization
  • Lots of duplicated effort
  • Little of reusability

Key Question

Is there an opportunity to for passive collaboration to drive student and community digital history projects?

Interface-driven sites

Some advantages

  • almost zero startup cost
  • easy administrative tools
  • ubiquitous

Some drawbacks

  • complex interfaces for simple tasks
  • too much document abstraction
  • reusing components is hard
  • siloing

Static sites

Some advantages

  • file based
  • preservation
  • precision/customization
  • experimentation
  • modularity

Some disadvantages

  • non-trivial start-up costs
  • troubleshooting rabbit holes
  • ongoing commitment

GitHub Pages Features

  • free hosting
  • minimum overhead
  • ubiquitious and stable
  • direct file editing (+ offline) →
  • version control →
  • fundamentally collaborative
  • issue tracking →
    • did we talk about this?
    • why did we do that?
    • whatever happened with that?
  • constantly evolving →
